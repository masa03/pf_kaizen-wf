# 6. 評価ロジック・自動計算仕様

## 6.1 効果金額算出

改善分野ごとに実績値を入力し、分野に応じた単価または算出式で効果金額を自動計算する。合計はすべての分野の効果金額の総和とする。算出式は改善分野マスタに定義し、Power Apps側のSwitch関数で動的に処理する。

## 6.2 評価点数のスコアリング

課長・部長それぞれが4つの評価軸でスコアリングを行う。①効果については、効果金額から自動算出されるa値と、金額算定不可の場合の手動入力b値のいずれかを使用する。②〜④はラジオボタンで手動選択し、各選択肢にはあらかじめ定められた点数が対応する。

**【重要】表彰区分設定時の評価スキップ**

表彰区分（小集団 パール賞（3等）/小集団 銅賞（2等）/小集団 銀賞（1等））が申請時に設定されている場合、上長（課長・部長）は②独創性、③努力工夫、④応用範囲の評価を行う必要はない。①効果の評価も不要。褒賞金額は表彰区分に応じた金額（小集団パール賞:3,000円、小集団銅賞:5,000円、小集団銀賞:10,000円）が自動適用される。上長は申請内容を確認し、承認または差戻のみ行う。

## 6.3 等級判定フロー

素点合計 = ①(a or b) + ② + ③ + ④ を算出後、職能換算テーブルで調整済みスコアに変換する。最終的に表彰区分（改善提案/小集団）に応じた等級テーブルで等級を判定し、褒賞金額を確定する。褒賞金額が5,000円以上の場合、部長承認フローへ自動遷移する。表彰区分が設定されている場合はこの等級判定フローをスキップし、賞区分の金額をそのまま使用する。

**【重要】FinalRewardAmountの転記ルール:**
- 課長承認のみで完結する場合（< 5,000円）: 課長の評価データのRewardAmountをメインリストのFinalRewardAmountに転記
- 部長承認が必要な場合（≥ 5,000円）: 部長の評価データのRewardAmountをメインリストのFinalRewardAmountに転記（部長の評価で上書き）
- 課長の評価データは評価データリストにそのまま保持（削除・変更しない）

## 6.4 条件分岐ロジック

| 条件 | 分岐先 | 処理 |
|------|-------|------|
| 課長承認 AND 褒賞金額 < 5,000円 | 承認完了 | ステータス→承認済、課長の褒賞金額→FinalRewardAmount転記、申請者+課長へ完了メール |
| 課長承認 AND 褒賞金額 ≥ 5,000円 | 部長評価 | ステータス→部長評価中、部長通知 |
| 課長差戻 | 申請者修正 | ステータス→差戻、NG通知 |
| 部長承認 | 承認完了 | ステータス→承認済、部長の褒賞金額→FinalRewardAmount転記（上書き）、申請者+部長へ完了メール |
| 部長差戻 | 申請者修正 | ステータス→差戻、NG通知 |
| 表彰区分=小集団パール賞/小集団銅賞/小集団銀賞 | 表彰区分金額で決定（スコアリングスキップ） | 表彰区分マスタから金額取得。小集団パール賞:3,000/小集団銅賞:5,000/小集団銀賞:10,000 |
| 差戻→再提出 | 課長評価から再開 | ステータス→申請中に戻し課長へ再通知。前回評価データは参考値として保持 |
