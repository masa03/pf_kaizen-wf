# 5. Power Automate フロー設計

## 5.1 フロー一覧

> ※ シンプルプランでは承認完了時の通知は申請者と承認者のみ。

| No. | フロー名 | トリガー | 処理概要 |
|-----|---------|--------|---------|
| 1 | 申請通知フロー | Lists項目作成時（ステータス=申請中） | 課長へ承認依頼メール送信。課長=申請者の場合は部長へ送信 |
| 2 | 課長承認フロー | Lists項目変更時（課長評価完了） | 承認: 金額判定→部長通知 or 完了メール（申請者+課長） / 差戻: NG通知 |
| 3 | 部長承認フロー | Lists項目変更時（部長評価完了） | 承認: 申請者+部長へ完了メール / 差戻: NG通知 |
| ★4 | 取下げ通知フロー | Lists項目変更時（ステータス=取下げ） | 承認者へ取下げ通知メール |
| ★5 | リマインダーフロー | スケジュール（日次） | 承認期限超過の案件を承認者へ督促 |

## 5.2 申請通知フロー詳細

| ステップ | アクション | 詳細 |
|---------|-----------|------|
| 1 | トリガー: 項目が作成されたとき | 改善提案メインリスト、条件: ステータス=申請中 |
| 2 | 承認者取得 | 社員マスタから申請者GID（SonyID）でLookUpし、課長GID→課長メールを取得。課長本人フラグ=1（課長=申請者）の場合は部長GIDを取得 |
| 2a | 条件分岐: 課長=申請者? | 課長本人フラグ=1の場合→部長へ承認依頼メール（課長承認スキップ）、それ以外→課長へ承認依頼メール |
| 3 | メール送信 | 課長へ承認依頼メール（提案タイトル、申請者名、リンク付き） |

## 5.3 課長承認フロー詳細

| ステップ | アクション | 詳細 |
|---------|-----------|------|
| 1 | トリガー: 項目が変更されたとき | 評価データリスト、条件: 評価者種別=課長 AND 判定≠空 |
| 2 | 条件分岐: 判定=承認? | |
| 2a-1 | Yes → 褒賞金額チェック | 金額 ≥ 5,000円 かどうか |
| 2a-2 | ≥5,000 → 部長承認へ | ステータスを「部長評価中」に更新、社員マスタから部長メールを取得し通知 |
| 2a-3 | <5,000 → 承認完了 | ステータスを「承認済」に更新、課長の褒賞金額をメインリストのFinalRewardAmountに転記、申請者と課長へ完了メール |
| 2b | No → 差戻処理 | ステータスを「差戻」に更新、申請者へNG通知メール |

## 5.4 部長承認フロー詳細

| ステップ | アクション | 詳細 |
|---------|-----------|------|
| 1 | トリガー: 項目が変更されたとき | 評価データリスト、条件: 評価者種別=部長 AND 判定≠空 |
| 2 | 条件分岐: 判定=承認? | |
| 2a | Yes → 承認完了 | ステータスを「承認済」に更新、部長の褒賞金額をメインリストのFinalRewardAmountに転記（部長評価で上書き）、申請者と部長へ完了メール |
| 2b | No → 差戻処理 | ステータスを「差戻」に更新、申請者へNG通知メール |

## 5.5 メール テンプレート

| メール種別 | 宛先 | 件名 | 本文に含む情報 |
|-----------|------|------|-------------|
| 承認依頼 | 承認者 | 【改善提案】承認依頼: {テーマ} | 申請者名、テーマ、効果金額合計、Power Appsリンク |
| NG通知（差戻） | 申請者 | 【改善提案】差戻通知: {テーマ} | 差戻者名、コメント、修正・再提出用リンク |
| 承認完了 | 申請者 + 承認者 | 【改善提案】承認完了: {テーマ} | 等級、褒賞金額、評価コメント |
| ★取下げ通知 | 承認者 | 【改善提案】取下げ通知: {テーマ} | 申請者名、取下げ理由 |
| ★リマインダー | 承認者 | 【改善提案】承認期限超過: {テーマ} | 申請日、期限日、経過日数、Power Appsリンク |

## 5.6 メールリンクURL設定 `[v10.1]`

メールテンプレートに含まれるPower Appsリンクの設定仕様。各メールの用途に応じてリンク先画面とURLパラメータを使い分ける。

| テンプレート | リンクテキスト | リンク先画面 | URLパラメータ |
|-------------|--------------|-------------|-------------|
| 3-1_申請通知_承認依頼.html | 評価画面を開く | 評価画面 | `?RequestID={RequestID}&EvalType=課長` |
| 3-2_課長承認_部長へ承認依頼.html | 評価画面を開く | 評価画面 | `?RequestID={RequestID}&EvalType=部長` |
| 3-2_課長承認_承認完了.html | 申請内容を確認する | 閲覧画面 | `?RequestID={RequestID}` |
| 3-2_課長承認_差戻通知.html | 申請内容を修正する | 申請フォーム | `?RequestID={RequestID}&Mode=Edit` |
| 3-3_部長承認_承認完了.html | 申請内容を確認する | 閲覧画面 | `?RequestID={RequestID}` |
| 3-3_部長承認_差戻通知.html | 申請内容を修正する | 申請フォーム | `?RequestID={RequestID}&Mode=Edit` |

> **URL形式**: `https://apps.powerapps.com/play/{AppID}?{パラメータ}`
>
> **注意**: `{AppID}` はPower AppsアプリのGUID。アプリ公開後に確定するため、テンプレートのプレースホルダー `XXXXXXXX` をGUIDに置換する。パラメータの `{RequestID}` はPower Automateの動的コンテンツで差し込む。
