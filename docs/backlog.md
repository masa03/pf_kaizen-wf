### §5 下書き保存機能（優先度: 高）

申請フォームで入力途中の内容を一時保存（下書き保存）できる機能を追加する。

**基本ルール**

| 項目 | 仕様 |
|------|------|
| v1のステータス | 「下書き」は既に定義済み。保存UIがないだけ |
| 保存タイミング | 申請者が「下書き保存」ボタンを押下時 |
| 保存先 | 改善提案メインリスト（ステータス=下書き） |
| 提出ボタンとの関係 | 下書き保存 → 後から編集 → 提出ボタンで正式提出 |

**動作**
- 新規申請時: 下書き保存ボタン押下 → ステータス=「下書き」でメインリストにレコード作成。Power Automateの申請通知フローはトリガーされない（条件: ステータス=申請中）
- 下書き編集時: 既存の下書きレコードを上書き更新
- 差戻後の再編集時: 差戻状態のレコードを編集中に下書き保存も可能とするか要検討

**検討項目**
- 下書き一覧の表示場所: ホーム画面 or SharePointビュー
- 下書きの有効期限（一定期間後に自動削除するか）
- バリデーション: 下書き保存時は必須チェックをスキップ or 最低限のチェックのみ
- 改善メンバー・改善分野実績の下書き保存: 子リストも一緒に保存する必要がある

### §8 社員マスタサジェスト検索UI（優先度: 中）

改善メンバー追加・評価者変更など、社員マスタから人を選択するUI全般を「GID手入力」から「氏名サジェスト検索＋選択」方式に変更する。

**対象箇所**
- 改善メンバー追加UI（申請フォーム右カラム）
- 評価者変更UI（§2で追加予定）
- 回覧者追加UI（§3で追加予定）

**基本ルール**

| 項目 | 仕様 |
|------|------|
| 入力方式 | テキストボックスに氏名（部分一致）を入力 → サジェストリスト表示 → 選択 |
| 委任対策 | 社員マスタ15,000件のため、最低3文字入力後にフィルタ開始など制限が必要 |
| フォールバック | GID完全一致入力でも検索可能とする |

**検討項目**
- 委任制限回避方式（`Search`関数は委任非対応。コレクション事前読み込み or SharePoint REST API経由）
- サジェスト表示件数の上限（例: 最大10件）
- GID入力 or 氏名入力の判定ロジック

**備考**: §2（評価者変更）・§3（回覧者）の実装より先にこちらを実装し、共通UIパーツとして利用する。
