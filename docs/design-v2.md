## 1. 添付資料の多ファイル形式対応・容量表記

### 概要
現在は画像ファイル（改善前/改善後）のみ閲覧画面で直接表示しているが、PDF・PPT・Excel等の添付も可能にする。また、添付ファイルの最大容量を申請画面上に明記する。

### ファイルサイズ上限

| ファイル種別 | 最大容量 |
|-------------|---------|
| 画像ファイル（JPG, PNG等） | 15 MB |
| 画像以外（PDF, PPTX, XLSX, DOCX等） | 240 MB |

- 申請画面のファイルアップロードUI付近に上限値を注記として表示する
- アップロード時にファイルサイズをバリデーションし、超過時はエラーメッセージを表示する

### 検討項目
- 対応ファイル形式: PDF, PPTX, XLSX, DOCX, 画像（既存）
- 閲覧画面でのプレビュー表示方式
  - Office Online埋め込み（SharePoint iframe）
  - PDFビューア
  - 画像以外はリンク表示のみにするか、インラインプレビューまで対応するか
- 添付ファイル種別（FileCategory）の拡張が必要か
- Power Automateフローのファイルアップロード処理への影響

## 2. 評価者の変更機能

### 概要
申請フォームで評価者（1人目・2人目）を表示し、変更・削除できる機能を追加する。v1の「課長→部長」固定ルートを柔軟化する。

> 旧v2 §2「課長不在時の直接部長承認」と旧v2 §3「承認者表示・変更機能」を統合・再整理した。

### 基本ルール

| 項目 | 仕様 |
|------|------|
| 評価者数 | **2名固定**（v1と同じ。1人目＝課長相当、2人目＝部長相当） |
| デフォルト値 | 社員マスタから自動取得（1人目=課長GID、2人目=部長GID） |
| 1人目の変更 | 可能。社員マスタ検索で別の社員を指定 |
| 1人目の削除 | **可能**。削除した場合、2人目が第1評価者となる（v1の「課長不在時の直接部長承認」と同等） |
| 2人目の変更 | 可能。社員マスタ検索で別の社員を指定 |
| 2人目の削除 | **不可**。2人目は必須 |

### 1人目削除時の動作
- 社員マスタの ManagerGID が空欄の場合も同様に1人目なしとして扱う（自動スキップ）
- ステータス遷移: 「申請中」→「回覧中」（回覧者ありの場合）or「部長評価中」に直接遷移
- 評価データ: 1人目のレコードは作成しない
- 褒賞金額の判定: 2人目が第1評価者として全金額を承認。金額による条件分岐（≥5,000円で部長承認）は不要
- 評価画面: 2人目が最初から評価する場合、1人目デフォルト値は空

### 検討項目
- 変更UI: 社員マスタ検索（GID入力 or 氏名検索）→ 選択
- バリデーション
  - 自分自身を評価者に設定できないようにするか
  - 変更先の社員のロール制約（課長/部長ロールを持っている必要があるか）
- 変更理由の入力（任意 or 必須）
- 変更履歴の記録（承認履歴リストに記録するか）
- 閲覧画面: 1人目なしの場合の表示（「1人目なし（2人目が直接評価）」等）
- Power Automateフローへの影響（評価者メール取得元をメインリストの承認者列から取得する設計に統一）

## 3. 回覧者（事前確認者）

### 概要
評価者に回る前に、申請内容を事前確認する「回覧者」を追加できる機能。回覧者は評価（スコアリング）は行わず、承認/差戻のみを行う。

### 基本ルール

| 項目 | 仕様 |
|------|------|
| デフォルト | なし（0人） |
| 追加 | 可能。申請フォームで社員マスタから検索して追加 |
| 人数上限 | 要検討（例: 最大5名） |
| 回覧タイミング | 申請提出後、**1人目の評価者に回る前**に全回覧者へ通知 |
| 回覧完了条件 | 全回覧者が承認するまで評価フローに進まない |

### 回覧者のアクション

| アクション | コメント | 後続処理 |
|-----------|---------|---------|
| **承認** | なし | 全回覧者が承認 → 1人目の評価者へ進む |
| **差戻** | **必須**（差戻理由を入力） | **即座に**申請者へ差戻通知メール（コメントをメール本文に記載）。残りの回覧者の回答は待たない |

### コメントの扱い
- 差戻コメントは**メール本文に記載するのみ**。SharePointリストへの永続保存は不要
- 実装上はPower Appsからの入力値をPower Automateフロー内で受け取り、メールテンプレートに差し込む

### ステータス遷移への影響

v1のステータスに「回覧中」を追加:

```
下書き → 申請中 → 回覧中（※回覧者ありの場合のみ）→ 課長評価中 → 部長評価中 → 承認済
                                                                          ↑ 差戻 ↓
                                                                          ← 差戻 ←
```

- 回覧者がいない場合: v1と同じフロー（回覧中ステータスをスキップ）
- 回覧者の差戻: ステータスを「差戻」に更新、申請者へ通知

### 結果通知メールへの追加
- 承認完了メール（課長承認完了・部長承認完了）の**Cc**に回覧者全員を追加
- v1の通知先: 申請者＋承認者 → v2: 申請者＋承認者＋回覧者（Cc）

### 検討項目
- 回覧者の入力UI: 申請フォームのどこに配置するか（右カラム or 左カラム下部）
- 回覧者用画面: 閲覧画面＋承認/差戻ボタンの簡易画面、またはメールリンクからの専用画面
- 回覧者リストの設計: 新規SharePointリスト or 改善メンバーリストの拡張
- 回覧者の回覧順序: 全員同時通知（並列）or 順番に回覧（直列）
- リマインダーの対象に回覧者も含めるか

## 4. 申請取消機能

### 概要
申請者が回覧中・評価待ちの申請を自ら取り消せる機能を追加する。v1の取下げ（下書き・申請中のみ）を拡張する。

### 取消可能なステータス

| ステータス | v1（取下げ） | v2（申請取消） |
|-----------|-------------|---------------|
| 下書き | ○ | ○ |
| 申請中 | ○ | ○ |
| 回覧中 | — | **○（新規）** |
| 課長評価中 | × | **○（新規）** |
| 部長評価中 | × | **○（新規）** |
| 承認済 | × | × |
| 差戻 | × | ×（再編集可能なため不要） |

### 取消時の処理
- ステータスを「取下げ」に更新
- 現在の承認者（回覧者/評価者）へ取消通知メールを送信
- 既に入力済みの評価データがある場合: **保持**（削除しない。参考記録として残す）

### 検討項目
- 取消ボタンの配置: 閲覧画面 or ホーム画面の申請一覧
- 取消理由の入力（任意 or 必須）
- 取消後の再提出可否（差戻と同様に再編集→再提出できるか）

## 5. 下書き保存機能

### 概要
申請フォームで入力途中の内容を一時保存（下書き保存）できる機能を追加する。

### 基本ルール

| 項目 | 仕様 |
|------|------|
| v1のステータス | 「下書き」は既に定義済み。保存UIがないだけ |
| 保存タイミング | 申請者が「下書き保存」ボタンを押下時 |
| 保存先 | 改善提案メインリスト（ステータス=下書き） |
| 提出ボタンとの関係 | 下書き保存 → 後から編集 → 提出ボタンで正式提出 |

### 動作
- 新規申請時: 下書き保存ボタン押下 → ステータス=「下書き」でメインリストにレコード作成。Power Automateの申請通知フローはトリガーされない（条件: ステータス=申請中）
- 下書き編集時: 既存の下書きレコードを上書き更新
- 差戻後の再編集時: 差戻状態のレコードを編集中に下書き保存も可能とするか要検討

### 検討項目
- 下書き一覧の表示場所: ホーム画面 or SharePointビュー
- 下書きの有効期限（一定期間後に自動削除するか）
- バリデーション: 下書き保存時は必須チェックをスキップ or 最低限のチェックのみ
- 改善メンバー・改善分野実績の下書き保存: 子リストも一緒に保存する必要がある

## 6. 集計・CSVダウンロード機能

### 概要
給与振込のために、承認済の改善提案データをCSV形式でダウンロードする機能。

### 出力項目（確認中）
- 褒賞金額（FinalRewardAmount）
- 申請者氏名（ApplicantName）
- 申請者GID（ApplicantGID）
- 等級（Grade）— 評価データから取得
- 部署（Department / Division / Bu / Section）
- 改善テーマ（Theme）

### 検討項目
- 実装方式
  - Power Apps内ボタン → Power Automateフローでcsv生成 → ダウンロード
  - SharePointリストのExportToExcel機能活用
  - 管理者画面（2-10）に組み込むか、独立機能にするか
- フィルタ条件
  - 期間指定（年度、月次等）
  - ステータス=承認済のみ
- メンバー単位の出力（1提案に複数メンバーがいる場合、メンバーごとに行を分けるか）
- 出向者・派遣社員の集計区分
- アクセス権限（管理者のみ or 部長以上）

## 7. 提案プラン（v1からの繰り越し）

### 7-1. ホーム画面（タスク 2-9）
- 自分の申請一覧ギャラリー
- 承認待ち一覧ギャラリー（課長/部長ロールに応じて表示）
- ステータスフィルタ / 期間フィルタ
- 各行タップで閲覧画面 or 評価画面に遷移

### 7-2. 管理者画面（タスク 2-10）
- マスタCRUD（社員マスタ・改善分野マスタ・表彰区分マスタ）
- 全件閲覧
- エクスポート機能（CSVダウンロード機能 6. と統合検討）

### 7-3. 取下げ通知フロー（タスク 3-5）
- 取下げ時の承認者通知メール
- v2の申請取消機能（§4）により、取消可能ステータスが拡張される

### 7-4. リマインダーフロー（タスク 3-6）
- **頻度: 毎朝1回**（日次スケジュールトリガー）
- 承認期限超過の案件を検出（承認期限5日超過）
- 承認者へ督促メール送信
- 回覧者への督促も対象に含めるか要検討（§3）

### 7-5. 承認履歴リスト（タスク 1-8）
- 申請/承認/差戻/取下げ/再提出/回覧承認/回覧差戻のアクションログ記録
