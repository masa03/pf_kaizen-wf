## 改善提案システム — 機能概要

**基盤**: SharePoint Lists + Power Apps + Power Automate ｜ **利用規模**: 15,000人 ｜ **作成日**: 2026年4月19日

---

### システム概要

業務改善提案の申請から評価・承認までのプロセスを電子化し、SharePoint + Power Platform 上で一元管理するシステム。

表彰区分は「改善提案」（4軸評価で等級判定、褒賞金額300〜10,000円）と「小集団賞」（パール賞3,000円/銅賞5,000円/銀賞10,000円の固定金額、スコアリング不要）の4区分。

---

### 申請画面

- **提案入力** — 表彰区分・改善テーマ・問題点・改善内容・改善完了日を入力
- **所属自動入力** — ログインユーザーのGIDから社員マスタを参照し、TEC/部門/部/課を自動表示
- **改善分野・効果金額自動算出** — 改善分野（14分野）を選択し実績値を入力 → 換算単価に基づき効果金額を自動算出
- **添付ファイル** — 複数ファイル対応（PDF/PPTX/XLSX/DOCX等）、改善前/改善後/その他の種別を選択
- **メンバー追加** — 氏名 or GIDのサジェスト検索で改善メンバーを追加（最大10名）
- **評価者指定** — 1人目（課長相当）・2人目（部長相当）を指定。1人目は省略可（部長へ直接遷移）
- **回覧者指定** — 承認前に内容を確認する回覧者を順序付きで設定（承認/差戻が可能）
- **下書き保存** — 入力途中の内容を保存し、後から編集・提出が可能
- **プレビュー** — 提出前に閲覧画面で申請内容を確認

### 評価画面（課長 / 部長 / 回覧者 共通）

- **申請内容表示** — 画面上部に申請内容（改善前後画像含む）を読み取り専用で表示
- **4軸スコアリング** — ①効果 ②独創性 ③努力工夫 ④応用範囲（各4段階×左右2択の8択ラジオボタン）
- **報奨金自動計算** — 素点合計 → 職能換算 → 等級判定 → 褒賞金額を自動算出
- **金額分岐** — 褒賞金額5,000円以上は部長承認へ自動遷移、未満は課長承認で完了
- **部長デフォルト値** — 部長評価時に課長の評価データを初期値表示。変更なしでそのまま承認可
- **表彰区分スキップ** — 小集団賞はスコアリング非表示、区分に応じた固定金額を自動適用
- **回覧者モード** — スコアリング非表示、承認/差戻 + コメント入力のみ
- **承認 / 差戻** — 差戻時は申請者に通知、修正・再提出が可能

### 閲覧画面

- **申請内容確認** — 提案の全項目を読み取り専用で表示（改善前/改善後の画像埋め込み対応）
- **改善実績一覧** — 分野ごとの実績値・換算単価・効果金額を一覧表示
- **評価結果確認** — 課長/部長の評価点数・等級・褒賞金額を表示（**申請者本人のみ**閲覧可）
- **取消操作** — 回覧中・評価中の申請を取り下げ可能（承認者・回覧者へ通知）
- **編集遷移** — 下書き・差戻・取下げ状態の申請を編集画面へ遷移

### 承認ワークフロー（Power Automate 10フロー）

| カテゴリ | 内容 |
|----------|------|
| 回覧 | 指定順に回覧者へ依頼 → 全員承認で評価者へ自動遷移。差戻時は申請者に通知 |
| 承認 | 課長評価 → 金額判定 → 部長評価（必要時）→ 承認完了。各ステップでメール通知 |
| 下書き・再提出 | 下書き/差戻/取下げから再提出時、同一フローで評価者へ通知 |
| 取下げ | 申請者が取消 → 承認者・回覧者へ取下げ通知メール |
| リマインダー | 日次9:00に滞留案件の担当者へ督促メール自動送信 |
| 添付ファイル管理 | ステージング転送（即時）+ 未提出分の自動クリーンアップ（日次） |

### 申請状況確認

- **SharePointリストビュー** — リンク付きカスタムビュー、申請一覧からPower Appsへ直接遷移
### 権限設計（2層構造）

| 層 | 仕組み | 内容 |
|----|--------|------|
| 第1層 | SharePoint WriteSecurity=2 | 自分が作成したアイテムのみ編集可（プラットフォームレベルで改ざん防止） |
| 第1.5層 | ナビゲーション非表示 | トランザクション系リストをサイトメニューから隠蔽（カジュアルアクセス抑止） |
| 第2層 | Power Apps UI制御 | ロール別の画面表示制御 + 評価者本人チェック（なりすまし防止） |

| 対象 | 権限ルール |
|------|-----------|
| 申請者 | 自分の申請のみ作成・編集・閲覧 |
| 課長 | 申請を閲覧・評価・承認/差戻 |
| 部長 | 褒賞金額5,000円以上の申請を閲覧・再評価・承認/差戻 |
| 管理者 | 全件閲覧・マスタメンテナンス（マスタ編集は管理者のみ） |


## データ構成（SharePoint Lists 8リスト）

| リスト | 用途 |
|--------|------|
| 改善提案メイン | 提案の基本情報・ステータス管理 |
| 改善メンバー | 提案ごとのメンバー情報（最大10名） |
| 改善分野実績 | 分野別の実績値・効果金額 |
| 評価データ | 課長・部長の評価結果・褒賞金額 |
| 回覧メンバー | 回覧者の順序・承認ステータス |
| 社員マスタ | 社員番号・氏名・組織・承認者情報（15,000人） |
| 改善分野マスタ | 改善分野の選択肢・単位・算出式（14分野） |
| 表彰区分マスタ | 表彰区分の選択肢・褒賞金額（4件） |
