改善WF_ステージングクリーンアップ 構築手順

フロー名: 改善WF_ステージングクリーンアップ

用途: 提出・下書き保存されずに放置された添付ファイルステージングレコードを48時間後に自動削除（§1）

各式の「コピー」ボタンをクリックすると、改行なしでクリップボードにコピーされます。

フロー概要

[繰り返し（Recurrence）トリガー]  毎日1回
    │
    ▼
[複数の項目の取得（Get items）]
    対象: 添付ファイルステージング
    フィルタ: Created lt '48時間前の日時'
    │
    ▼
[それぞれに適用する（Apply to each）]
    └─ [項目の削除（Delete item）]  ステージングレコード削除

Step 0: フロー作成

  1. Power Automate → ＋ 作成 → スケジュール済みクラウドフロー
  2. フロー名: 改善WF_ステージングクリーンアップ
  3. 繰り返し間隔: 1日
  4. 開始時刻: 任意（例: 深夜 03:00）
  5. 「作成」をクリック
このフローはスケジュール実行のため、Power Appsとの接続は不要です。

Step 1: トリガー設定（繰り返し / Recurrence）

トリガー「繰り返し（Recurrence）」カードの設定を確認する。

Step 2: 複数の項目の取得（期限切れステージングレコード）

  1. ＋ 新しいステップ → 複数の項目の取得（Get items）
  2. アクション名を 期限切れステージング取得 に変更
サイトのアドレス

対象のSharePointサイトを選択

リスト名

添付ファイルステージング

フィルター クエリ
Created lt '@{addHours(utcNow(), -48)}'
上から順に取得

500

⚠️ フィルタークエリの addHours(utcNow(), -48) 部分は式タブ（fx）で入力すること。テキスト欄に直接入力すると文字列リテラルとして扱われる。
ODataフィルタ: 作成日時が現在から48時間以上前のレコードのみ取得する。正常にSubmitされたレコードはフローで削除済みのため、ここで取得されるのは放置されたゴミレコードのみ。

Step 3: それぞれに適用する（ステージングレコード削除）

  1. ＋ 新しいステップ → それぞれに適用する（Apply to each）
  2. アクション名を 期限切れレコード削除ループ に変更
以前の手順から出力を選択
@{body('期限切れステージング取得')?['value']}
⚠️ 動的コンテンツパネルで「期限切れステージング取得」の value を選択してもよい。
ループ内アクション

3-1. 項目の削除（Delete item）

  1. ループ内で アクションの追加 → 項目の削除（Delete item）
  2. アクション名を ステージングレコード削除 に変更
サイトのアドレス

対象のSharePointサイトを選択

リスト名

添付ファイルステージング

ID
@{items('期限切れレコード削除ループ')?['ID']}
SPリスト項目を削除すると、添付ファイル（ステージングに保存されたファイル）も自動的に削除される。別途ファイル削除アクションは不要。

Step 4: テスト・有効化

  1. フローを保存
  2. テスト → 手動 → 「テスト」をクリック
  3. 実行履歴で各アクションが成功していることを確認
  4. SharePoint「添付ファイルステージング」リストを開き、48時間以上前のレコードが削除されていることを確認
  5. フローがオンになっていることを確認（スケジュール実行が有効になる）
⚠️ テスト時に48時間以上前のレコードが存在しない場合、「それぞれに適用する」は0回実行される（エラーにはならない）。テスト用に古いレコードを手動作成してからテストすること。
