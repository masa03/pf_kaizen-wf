マスタデータ インポート手順

用途: SharePoint リスト作成後に、マスタデータ（改善分野・表彰区分・社員）をExcelからインポートする

前提: リスト自動作成フロー（ui-manual-sharepoint.html）が完了済みであること

各マスタの投入データをExcelに転記し、グリッドビューで貼り付けるだけで完了します。

インポート対象サマリー

No.リスト名件数方法
1改善分野マスタ14件グリッドビューで貼り付け
2表彰区分マスタ4件グリッドビューで貼り付け
3社員マスタテスト: 50件 / 本番: 別途Excelからコピー&ペースト
なぜExcelインポートか: マスタデータは少量かつ低頻度の投入のため、Power Automate REST APIで構築するのはオーバーエンジニアリング。SharePoint UIの「グリッドビューで編集」機能でExcelから直接貼り付けるのが最も効率的。

共通手順: グリッドビュー貼り付けの操作方法

  1. SharePoint サイトで対象のリストを開く
  2. コマンドバーの 「グリッドビューで編集」 をクリック
  3. 下記の投入データ表をExcelにコピーし、ヘッダー行を除いたデータ行のみを選択してコピー（Ctrl+C）
  4. グリッドビューの最初の空行の Title列（左端）をクリックして選択
  5. Ctrl+V で貼り付け
  6. 貼り付け後、「グリッドビューの終了」 をクリックして保存
列の順序に注意: グリッドビューの列順がこの手順書の表の列順と一致している必要があります。ビューの列順が異なる場合は、先にビューの列設定で順序を合わせるか、1列ずつ手動で入力してください。
はい/いいえ列: SharePoint の「はい/いいえ」列にExcelから貼り付ける場合、はい または いいえ と日本語で入力してください。True/False では正しく認識されない場合があります。

データソース

社員マスタのデータは環境によって異なります:

  • テスト環境: scripts/csv/test_employees.csv（50件）
  • 本番環境: 人事部門から提供される社員データ（15,000件規模）

インポート手順

  1. CSVファイルをExcelで開く（文字コードUTF-8に注意）
  2. 先頭に Title列 を追加し、GID列と同じ値を入れる
    （Excel数式: セルに =B2 を入れて下までコピー）
  3. True/False の列を はい/いいえ に変換する
    （対象列: IsManagement, IsDeptHead, IsDirector, IsManager, IsActive）
    （Excel置換: Ctrl+H → 検索「True」→ 置換「はい」、検索「False」→ 置換「いいえ」）
  4. SharePointの社員マスタリストを開き、「グリッドビューで編集」をクリック
  5. Excelでヘッダー行を除いたデータ行を全選択してコピー（Ctrl+C）
  6. グリッドビューの最初のTitle列セルをクリックし、Ctrl+V で貼り付け
  7. 「グリッドビューの終了」をクリックして保存
大量データの場合（本番15,000件）: グリッドビュー貼り付けでは一度に処理できる件数に制限がある場合があります。1,000件ずつに分割して貼り付けることを推奨します。エラーが発生した場合は、件数をさらに小さく（500件や200件）して試してください。
CSV列順とグリッドビュー列順の対応: CSVの列順（GID, EmployeeName, Email, ...）と、SharePointグリッドビューの列順が一致していない場合があります。事前にビューの列設定で順序を合わせてください。

投入後の確認

各マスタの投入後、以下を確認してください:

  1. 件数の確認: リストのアイテム数が想定どおりか
    • 改善分野マスタ: 14件
    • 表彰区分マスタ: 4件
    • 社員マスタ: CSVの行数と一致
  2. はい/いいえ列の確認: チェックボックスが正しく設定されているか（特に社員マスタの管理職フラグ・各本人フラグ）
  3. 選択肢列の確認: 改善分野マスタの「分野種別」が正しく選択されているか
  4. 数値列の確認: 金額換算単価（ConversionRate）の値が正しいか（特に活人: 760,232）
すべてのマスタデータが正しく投入されれば、Power Apps アプリケーションからマスタ参照が可能になります。
