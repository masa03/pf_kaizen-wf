SharePoint リスト権限設定 手順書

用途: 各リストのアイテムレベル権限（WriteSecurity）、マスタリストの権限降格、ナビゲーション非表示を設定する

前提: リスト作成・マスタデータ投入が完了済みであること。サイト所有者権限でログインしていること

設計書 セクション7（セキュリティ・権限設計）に基づく設定です。すべてSharePoint UIの操作のみで完了します。

設定サマリー

設定カテゴリ対象リスト操作内容
Step 1 WriteSecurity=2 改善提案メイン 改善メンバー 改善分野実績 評価データ 自分のアイテムのみ編集可に制限
Step 2 マスタ権限降格 社員マスタ 改善分野マスタ 表彰区分マスタ メンバーグループを「読み取り」のみに降格
Step 3 ナビゲーション非表示 改善メンバー 改善分野実績 評価データ 回覧メンバー 添付ファイル クイック起動（左ナビ）から非表示
権限制御の2層構造: SharePointのネイティブ権限（第1層）でデータ改ざんをプラットフォームレベルで防止し、Power Appsのアプリロジック（第2層）で画面表示を制御します。この手順書では第1層を設定します。

1 アイテムレベル権限（WriteSecurity=2）

トランザクションリスト4つに対して、「自分が作成したアイテムのみ編集可」を設定します。

目的: 第三者による申請データ・評価データの改ざんをSharePointプラットフォームレベルで防止する。Power Appsをバイパスしてリストを直接編集しても、他人のデータは変更できなくなります。

対象リストと設定値

リスト名読み取りアクセス作成/編集アクセス理由
改善提案メインすべてのアイテムユーザー本人のアイテム課長/部長が閲覧する必要があるため読み取りは全件
改善メンバーすべてのアイテムユーザー本人のアイテム申請者がPatchで作成（Created By=申請者）
改善分野実績すべてのアイテムユーザー本人のアイテム申請者がPatchで作成（Created By=申請者）
評価データすべてのアイテムユーザー本人のアイテム評価者がPatchで作成。報奨金に直結するデータ

操作手順（リストごとに繰り返し）

  1. 対象リストを開く
  2. 右上の ⚙（歯車アイコン） → 「リストの設定」 をクリック
  3. 「詳細設定」 をクリック
  4. 「読み取りアクセス」セクション:
    → 「すべてのアイテム」 を選択
  5. 「アイテムの作成と編集のアクセス権」セクション:
    → 「ユーザー本人のアイテム」 を選択
  6. 「OK」 をクリックして保存

チェックリスト

改善提案メイン — 設定完了
改善メンバー — 設定完了
改善分野実績 — 設定完了
評価データ — 設定完了
テスト環境での注意: テスト環境ではアカウントが1つのため、WriteSecurity=2 を設定すると自分が作成したアイテムしか編集できなくなります。テスト中は課長・部長ロールでの評価テストに支障が出る場合があるため、テストフェーズが完了してから本設定を適用することを推奨します。

2 マスタリストの権限降格（読み取りのみ）

マスタリスト3つに対して、メンバーグループの編集権限を「読み取り」のみに降格します。管理者（サイト所有者）のみがマスタデータを編集できるようにします。

目的: 一般ユーザーがSharePointリストを直接開いてマスタデータ（社員情報・分野設定・表彰区分）を変更できないようにする。Power Apps側でもマスタは参照のみだが、SPレベルでも書き込みを防止します。

対象リスト

リスト名変更前変更後
社員マスタメンバー: 編集可（サイト継承）メンバー: 閲覧のみ
改善分野マスタメンバー: 編集可（サイト継承）メンバー: 閲覧のみ
表彰区分マスタメンバー: 編集可（サイト継承）メンバー: 閲覧のみ

操作手順（リストごとに繰り返し）

  1. 対象リストを開く
  2. 右上の ⚙（歯車アイコン） → 「リストの設定」 をクリック
  3. 「権限と管理」セクションの 「このリストに対する権限」 をクリック
  4. リボン（上部バー）の 「権限の継承を中止」 をクリック
    → 確認ダイアログが出たら「OK」をクリック（親サイトの権限がコピーされた状態で独立します）
  5. メンバーグループ（例: kaizen-wf のメンバー）の左のチェックボックスにチェック
  6. リボンの 「ユーザー権限の編集」 をクリック
  7. 現在チェックされている権限レベル（「編集」や「投稿」など）をすべて外す
  8. 「閲覧」 にのみチェックを入れる
  9. 「OK」 をクリックして保存

チェックリスト

社員マスタ — 権限降格完了
改善分野マスタ — 権限降格完了
表彰区分マスタ — 権限降格完了
重要: マスタデータのインポート（ui-manual-sharepoint-import.html）が完了してからこの手順を実施してください。権限降格後は、メンバーグループのユーザーではマスタデータの編集ができなくなります。
グループ名について: メンバーグループの名前はサイト作成時の設定によって異なります（例: 「kaizen-wf のメンバー」「サイト名 Members」など）。リストの権限ページで表示されるグループ名を確認してください。

3 サイトナビゲーションからリストを非表示

管理者が直接参照する必要がないリストをクイック起動（左ナビゲーション）から非表示にし、一般ユーザーのカジュアルアクセスを抑止します。

目的: WriteSecurity=2 は他人のデータ改ざんを防止しますが、本人がSPリストを直接開いてステータスを無視して編集することは防げません。ナビゲーションから非表示にすることで、リストの存在自体を目立たなくします。URL直打ちでのアクセスは引き続き可能なため、管理者のトラブルシューティングには支障ありません。

表示/非表示の設計

リスト名ナビゲーション理由
改善提案メイン表示管理者の申請一覧確認で使用
社員マスタ表示管理者がメンテナンスで直接編集
改善分野マスタ表示管理者がメンテナンスで参照
表彰区分マスタ表示管理者がメンテナンスで参照
改善メンバー非表示Power Apps経由でのみ操作
改善分野実績非表示Power Apps経由でのみ操作
評価データ非表示Power Apps経由でのみ操作。報奨金に直結
回覧メンバー非表示Power Apps経由でのみ操作
添付ファイル非表示Power Apps経由でのみ参照

操作手順（非表示にするリストごとに繰り返し）

  1. 対象リストを開く
  2. 右上の ⚙（歯車アイコン） → 「リストの設定」 をクリック
  3. 「詳細設定」 をクリック
  4. 「クイック起動にこのリストを表示しますか?」セクション:
    → 「いいえ」 を選択
  5. 「OK」 をクリックして保存

設定後の確認

WriteSecurity の確認

  1. 一般ユーザー（メンバーグループ）のアカウントでサイトにアクセス
  2. 改善提案メインリストを開く
  3. 他のユーザーが作成したアイテムをクリック → 編集ができないことを確認
  4. 自分が作成したアイテムをクリック → 編集ができることを確認
テスト環境（アカウント1つ）での確認方法: テスト環境ではすべてのアイテムのCreated Byが同一アカウントになるため、WriteSecurity=2 の効果を直接確認できません。設定値が正しいかは「リストの設定 → 詳細設定」で読み取りアクセス/編集アクセスの選択状態を目視確認してください。

マスタ権限降格の確認

  1. 一般ユーザーのアカウントで社員マスタを開く
  2. 「グリッドビューで編集」や「新規」ボタンが表示されないことを確認
  3. アイテムをクリックしても編集できないことを確認
  4. サイト所有者のアカウントでは引き続き編集できることを確認

ナビゲーション非表示の確認

  1. サイトのトップページを開く
  2. 左ナビゲーションに以下のリストが表示されていないことを確認:
    改善メンバー 改善分野実績 評価データ 回覧メンバー 添付ファイル
  3. 以下のリストは表示されていることを確認:
    改善提案メイン 社員マスタ 改善分野マスタ 表彰区分マスタ
すべての権限設定が完了しました。SharePointプラットフォームレベルでのデータ保護が有効になり、Power Appsアプリケーションのセキュリティと合わせて2層の権限制御が機能します。
