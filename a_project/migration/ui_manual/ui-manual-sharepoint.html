SharePoint リスト自動作成フロー 構築手順

用途: PnP PowerShell が使用できない環境で、Power Automate の REST API により全リスト・列・インデックスを一括作成する

前提: 対象 SharePoint サイトの管理者権限があること（Azure AD アプリ登録は不要）

各式の「コピー」ボタンをクリックすると、改行なしでクリップボードにコピーされます。

構成サマリー

項目数量
フロー数1
アクション数14（トリガー1 + 作成3 + Apply to each 5 + HTTP要求 5）
実行される HTTP リクエスト総数253回（リスト11 + Title非表示11 + 列作成108 + 列名変更108 + インデックス15）

使用 JSON ファイル

ファイル内容件数
01_create-lists.json10リスト + 添付ファイルライブラリ作成11件
02_all-columns.json全リストの列定義（listName + DisplayTitle 付き）108列
13_indexes.jsonインデックス設定15件

02_all-columns.json の構造

列定義 JSON は以下の構造になっている。Title は英語の内部名（InternalName になる）、DisplayTitle は日本語の表示名。 listName と DisplayTitle は SharePoint に送信する前に除去する。
プロパティ用途例
listNameURI のルーティング用（除去して送信）"改善提案メイン"
Title英語内部名（InternalName になる）"RequestID"
DisplayTitle日本語表示名（Phase 2b で設定）"リクエストID"

3HTTP 要求アクション追加（②の内部）

  1. Apply to each の内部で + → アクションの追加 → 検索: 「SharePoint HTTP」 → 「SharePoint に HTTP 要求を送信します」を選択
サイトのアドレス ドロップダウンから対象 SharePoint サイトを選択
方法（メソッド） POST
URI
_api/web/lists
ヘッダー
{"Accept": "application/json;odata=verbose", "Content-Type": "application/json;odata=verbose"}

ボディ:

{ "__metadata": {"type": "SP.List"}, "AllowContentTypes": true, "BaseTemplate": @{items('Apply_to_each_-_create_lists')?['BaseTemplate']}, "ContentTypesEnabled": false, "Title": "@{items('Apply_to_each_-_create_lists')?['Title']}" }
ボディの入力方法: ボディフィールドをクリックして直接入力する。@{items(...)} の部分は Power Automate が自動的に動的コンテンツとして認識する。
BaseTemplate は数値のため " で囲まない。Title は文字列のため " で囲む。

5HTTP 要求アクション追加（④の内部）

サイトのアドレス ドロップダウンから対象 SharePoint サイトを選択
方法（メソッド） POST
URI
_api/web/lists/GetByTitle('@{items('Apply_to_each_-_hide_title')?['Title']}')/fields/GetByInternalNameOrTitle('Title')
ヘッダー
{"Accept": "application/json;odata=verbose", "Content-Type": "application/json;odata=verbose", "X-HTTP-Method": "MERGE", "If-Match": "*"}
ボディ

01_create-lists.json の各エントリには Title（リスト名）があるため、URI でそのままリスト名として使用できる。
ボディは全リスト共通の固定値。メソッドは POST + ヘッダー X-HTTP-Method: MERGE で更新操作になる。

8HTTP 要求アクション追加（⑦の内部）

  1. Apply to each の内部で + → アクションの追加 → 「SharePoint に HTTP 要求を送信します」
サイトのアドレス ドロップダウンから対象 SharePoint サイトを選択
方法（メソッド） POST
URI
_api/web/lists/GetByTitle('@{items('Apply_to_each_-_add_columns')?['listName']}')/fields
ヘッダー
{"Accept": "application/json;odata=verbose", "Content-Type": "application/json;odata=verbose"}
ボディ（式タブ）
removeProperty(removeProperty(items('Apply_to_each_-_add_columns'), 'listName'), 'DisplayTitle')
ボディの入力方法: ボディフィールドをクリック → 「式」タブ → 上記の式を入力 → OK。
入力後、ボディに紫色のタグが表示されればOK。テキストのまま表示されている場合は式として認識されていないので、再度「式」タブから入力し直す。
removeProperty() を2重にすることで listName と DisplayTitle の両方を除いて送信する。

10HTTP 要求アクション追加（⑨の内部）

サイトのアドレス ドロップダウンから対象 SharePoint サイトを選択
方法（メソッド） POST
URI
_api/web/lists/GetByTitle('@{items('Apply_to_each_-_rename_columns')?['listName']}')/fields/GetByInternalNameOrTitle('@{items('Apply_to_each_-_rename_columns')?['Title']}')
ヘッダー
{"Accept": "application/json;odata=verbose", "Content-Type": "application/json;odata=verbose", "X-HTTP-Method": "MERGE", "If-Match": "*"}

ボディ:

{"__metadata": {"type": "SP.Field"}, "Title": "@{items('Apply_to_each_-_rename_columns')?['DisplayTitle']}"}
メソッドとヘッダーについて: メソッドは POST を選択するが、ヘッダーに X-HTTP-Method: MERGE を設定することで更新（PATCH）操作として動作する。
URI で Title（英語内部名）を指定して対象列を特定し、ボディで Title を DisplayTitle（日本語表示名）に書き換える。
ボディの入力方法: ボディフィールドに直接テキストとして入力する（式タブは不要）。
@{items(...)} の部分は Power Automate が自動的に動的コンテンツとして認識する。

フロー実行

  1. フローを保存
  2. テスト → 手動 → テストの実行
  3. 全253リクエストの実行に数分〜十数分かかる。完了まで待つ
  4. 各 Phase の Apply to each が緑チェックになることを確認
  5. SharePoint サイトの「サイトコンテンツ」で全リストが作成されていることを確認

確認チェックリスト

確認項目期待値
サイトコンテンツのリスト数10リスト + 1ドキュメントライブラリ
改善提案メインの列数23列（+ Title列）
列の内部名英語（例: RequestID, ApplicantEmail）
列の表示名日本語（例: リクエストID, 申請者メール）
改善提案メインのステータス列の選択肢下書き / 申請中 / 回覧中 / 課長評価中 / 部長評価中 / 承認済 / 差戻 / 取下げ
Title 列の状態全リストで非表示（Hidden=true, Required=false）

列の確認方法（REST API）

テスト用に別の「SharePoint に HTTP 要求を送信します」アクションを追加し、以下を実行:

方法 GET
URI
_api/web/lists/GetByTitle('改善提案メイン')/fields?$filter=CanBeDeleted eq true&$select=Title,InternalName,TypeAsString,Indexed
レスポンスの InternalName が英語、Title が日本語になっていれば成功。

エラー時の対応

症状原因対処
Phase 1 で一部リストが失敗同名リストが既に存在既存リストを削除してから再実行
Phase 2a で一部列が失敗同名列が既に存在該当列を削除して再実行、または無視（既存列はそのまま使える）
Phase 2a ボディで「無効な JSON」エラー式がテキストとして入力されているボディの「式」タブから入力し直す。紫色タグが表示されればOK
Phase 2b で列が見つからないPhase 2a が完了していないPhase 2a の成功を確認してから再実行
「サイトのアドレス」で対象サイトが出ないSharePoint コネクタの接続先が異なるコネクタの接続を追加し、対象テナントのアカウントで認証する
フローが終わらない・キャンセルできないエラーによるリトライが連続しているマイフロー → フロー名の右「...」→「オフにする」で強制停止

実行後の追加作業（UI で実施）

以下は REST API で自動化が困難なため、SharePoint UI で手動設定する。

1. 添付ファイル有効化

対象: 添付ファイルステージング

  1. リストを開く → 右上 歯車アイコン → リストの設定
  2. 詳細設定 → 添付ファイル → 有効 → OK
Title 列の非表示は 02_all-columns.json の "action": "update" エントリにより Phase 2a で自動処理されるため、手動設定は不要です。
これで全リスト・列・インデックスの作成が完了です。次のステップ（マスタデータ投入）に進んでください。

フローの削除

このフローはリスト作成のための一時的なフローです。リスト作成完了後は削除して構いません。

  1. Power Automate → マイフロー
  2. kaizen-create-lists の右側 「...」 → 削除
